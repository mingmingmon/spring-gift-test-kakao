# step1 학습 내용
1. "행동 기반 인수 테스트"란 무엇인가?
    - 시스템 경계인 API에서 사용자 시나리오를 실행하고 최종적으로 어떤 결과가 발생핬는지 테스트 하는 것.
    - 구현 기반인 단위 테스트와 비교한다면, 구현기반의 경우는 어떤 메소드를 호출했는지에 집중. 
    - 행동기반 인수테스트는 재고가 실제로 1 줄었는지, 주문 생성이 완료 되었는지 확인.
2. "외부행동"이란 무엇인가?
    - 시스템 밖에서 관찰 가능한 결과.
    - 일반적으로 HTTP 응답, 예외 타입, DB 상태, 비즈니스 결과로 확인 가능.
3. 왜 "API 경계"에서 테스트해야 하는가?
    - UI가 바뀌어도 확인 용이, 모든 직군이 같은 언어(요구사항-계약)로 이해 가능.
4. 단위 테스트의 "착시"가 무엇이길래 인수테스트가 필요한가?
    - verify(option).decrease(3);와 같은 일반적이 단위테스트의 경우 
    - 메소드 호출 여부에 관심이 있기 때문에 응답 형태가 바뀌거나, 예외 타입이 바뀌는 경우에 대해서는 발견하기 어려움.
5. RestAssured로 인스테스트를 어떻게 사용할 수 있는가?
    - Given(준비) : 테스트 데이터 초기 상태 세팅
    - When(행동) : API 호출 (ResetAssured)
    - Then(검증) : HTTP 응답 + 후속 행동으로 상태 변화 확인
6. InventoryService 리팩토링이란 무엇인가?
    - 단위 테스트는 리팩토링 과정에서 실패할 수 있지만 인수 테스트는 통과한다.
    - OrderService
      └── inventory.decrease()
    - OrderService
      └── InventoryService
      └── inventoryRepository
    - 위의 예시에서 구조가 바뀌면 단위 테스트는 실패하지만 재고가 잘 줄어든다면 비즈니스에 해당하는 인수 테스트는 통과한다.
